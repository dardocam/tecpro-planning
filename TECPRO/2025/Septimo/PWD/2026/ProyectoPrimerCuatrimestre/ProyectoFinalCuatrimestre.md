# 🧩 Proyecto Final de Cuatrimestre – E-Commerce

## 👥 **Modalidad**

* Trabajo en **grupos de 2 estudiantes**
* Ambos deben participar activamente
* La evaluación será **individual + grupal**

❗ Durante la defensa cada integrante deberá explicar partes distintas del sistema

---

# 🎯 **Objetivo**

Desarrollar un sistema web tipo **E-Commerce en PHP + MySQL**, aplicando:

* Backend
* Base de datos
* Seguridad
* Lógica de negocio

---

# 🧱 **ORGANIZACIÓN DEL PROYECTO**

El proyecto se divide en:

## 🔹 Entrega 1 (Base del sistema)

## 🔹 Entrega 2 (Sistema funcional completo)

## 🔹 Entrega Final (Defensa + mejoras)

---

# 📆 **🔹 ENTREGA 1 – BASE DEL SISTEMA**

📅 Fecha: 29/5/2026

## 📌 Deben desarrollar:

### 🗄️ Base de datos

* usuarios
* productos
* carrito
* pedidos
* detalle_pedido

✔ Con claves primarias y foráneas

---

### 👤 Sistema de usuarios

* Registro
* Login
* Uso de sesión (`$_SESSION`)

---

### 🔐 Seguridad obligatoria

* `password_hash()`
* `password_verify()`

---

## 📤 Entregan:

* Archivo `.sql`
* Código funcionando (registro + login)
* Capturas o video corto

---

## 🎤 Defensa en clase (individual)

Cada integrante deberá:

* Explicar la base de datos
* Explicar el proceso de login

💡 Preguntas típicas:

* ¿Dónde se guarda la contraseña?
* ¿Qué hace la sesión?
* ¿Cómo se valida un usuario?

---

# 📆 **🔹 ENTREGA 2 – SISTEMA FUNCIONAL**

📅 Fecha: 29/6/2026

## 📌 Deben desarrollar:

### 🛒 E-Commerce básico

* Listado de productos
* Carrito de compras
* Confirmación de pedido
* Historial de compras

---

### 🔑 Roles y permisos

* Admin / Usuario / Visitante
* Restricción de acceso

---

### 🔎 Búsqueda y filtros

* Buscar por nombre
* Filtrar por categoría o precio

---

### 🧑‍💻 Panel admin

* Crear productos
* Editar
* Eliminar
* Ver pedidos

---

## 📤 Entregan:

* Proyecto funcionando completo
* Base de datos actualizada

---

## 🎤 Defensa en clase (individual)

Cada integrante debe:

* Explicar una funcionalidad distinta
* Mostrar el sistema en vivo

💡 Preguntas:

* ¿Cómo funciona el carrito?
* ¿Cómo se guarda un pedido?
* ¿Cómo restringís el acceso al admin?

---

# 📆 **🏁 ENTREGA FINAL – DEFENSA + MEJORAS**

📅 Fecha: A partir del 3/7/2026 al 17/7/2026

---

## 📌 Deben presentar:

### ✔ Proyecto completo funcionando

### ✔ Documentación

* Cómo funciona el sistema
* Estructura de base de datos
* Problemas encontrados

---

### ✔ Mejora obligatoria (mínimo 1)

Ejemplos:

* Mejor diseño visual
* Validaciones adicionales
* Control de stock
* Mensajes de usuario

---

## 🎤 DEFENSA FINAL (INDIVIDUAL)

Cada estudiante deberá:

### 🔹 Explicar:

* Parte del código (elegida por el docente)
* Flujo completo del sistema

### 🔹 Resolver en vivo:

Se pedirá una modificación simple, por ejemplo:

* Agregar validación
* Modificar una consulta
* Cambiar comportamiento del carrito

👉 Si no puede hacerlo → no aprueba

---
